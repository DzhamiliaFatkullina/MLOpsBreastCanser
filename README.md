# MLOps Demo — Breast Cancer Classifier

Учебный проект, демонстрирующий полный ML-пайплайн: подготовка данных, обучение модели, деплой через REST API и веб-интерфейс, всё упаковано в Docker.

Датасет — Breast Cancer Wisconsin (569 примеров, 30 фичей, бинарная классификация). Модель — Random Forest с предобработкой. Основной фокус проекта — структура пайплайна и деплой, а не качество модели.

## Структура

```
datasets/prepare.py            загрузка, очистка, train/test split
models/train_from_csv.py       обучение sklearn Pipeline → сохранение модели и метаданных
deployment/api/main.py         FastAPI: загружает модель при старте, отдаёт /predict
deployment/app/app.py          Streamlit UI: принимает фичи, вызывает API, показывает результат
deployment/docker-compose.yaml оркестрация сервисов
```

Модель сохраняется как полный sklearn **Pipeline** (StandardScaler + RandomForestClassifier одним объектом) — трансформация при инференсе гарантированно идентична той, что при обучении. В продакшене этот артефакт хранился бы в **model registry** (MLflow, Weights & Biases) с версией, метриками и ссылкой на породивший его эксперимент.

## Запуск

**1. Подготовить данные**
```bash
python code/datasets/prepare.py
```
Дропает NaN, убирает выбросы по z-score > 5, стратифицированный сплит 80/20, результат в `data/processed/`.

**2. Обучить модель**
```bash
python code/models/train_from_csv.py
```
Сохраняет `models/model.joblib` и `models/metadata.json` с именами фичей и точностью на тесте.

**3. Поднять сервисы**
```bash
docker compose -f code/deployment/docker-compose.yaml up --build
```

| Сервис | URL |
|---|---|
| API | http://localhost:8000 |
| UI  | http://localhost:8501 |

## API

`POST /predict` — принимает 30 флоатов в том же порядке что при обучении:
```json
{ "features": [17.99, 10.38, 122.8, ...] }
```
Возвращает предсказание, вероятности классов и их названия.

`GET /` — хелсчек, возвращает ожидаемое количество фичей.

## Ограничения и что было бы в продакшене

Проект охватывает базовый цикл, но намеренно не включает инфраструктуру, которая нужна в реальной системе:

- **Трекинг экспериментов** (MLflow, W&B) — логирование гиперпараметров, метрик и артефактов для каждого запуска
- **Версионирование данных** (DVC) — воспроизводимость на уровне данных, не только кода
- **CI/CD** — автоматический ретрейн и деплой при изменении данных или кода
- **Мониторинг** — отслеживание распределения предсказаний, алёрты на data drift
- **Инфраструктура** — хранилище артефактов (S3/GCS), Kubernetes, управление секретами
